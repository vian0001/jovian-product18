TOPOLOGI JARINGAN

Tampilan topologi di gns3

Topologi jaringan yang digunakan untuk implementasi DNS di MikroTik terdiri dari komponen-komponen berikut:

Router MikroTik berfungsi sebagai DNS server dan DNS cache untuk seluruh jaringan lokal. Semua klien dalam jaringan akan menggunakan router sebagai DNS resolver utama.

Berikut adalah komponen-komponen dalam topologi jaringan DNS:

1. MikroTik Router (192.168.88.1)
# Berfungsi sebagai DNS server dan DNS cache
2. Internet/ISP ‚Üí Sumber DNS Upstream
# Menyediakan resolusi DNS untuk domain publik (8.8.8.8, 1.1.1.1)
3. Jaringan LAN (192.168.88.0/24)
# Perangkat klien yang menggunakan layanan DNS MikroTik
4. Server Internal (192.168.88.254)
# Server internal yang di-resolve melalui Static DNS

Dalam topologi ini, MikroTik Router menangani semua permintaan DNS dari jaringan lokal. Untuk domain publik (seperti google.com), router akan meneruskan permintaan ke server DNS upstream (seperti 8.8.8.8). Untuk domain internal (seperti server.local), router akan menggunakan entri static DNS yang telah dikonfigurasi.

Router MikroTik harus dikonfigurasi dengan benar untuk membatasi akses DNS hanya untuk jaringan internal. Ini mencegah router digunakan sebagai open DNS resolver yang dapat dimanfaatkan untuk serangan DNS amplification.

Dengan menggunakan topologi ini, kita dapat mengoptimalkan kinerja DNS dengan memanfaatkan fitur static DNS dan DNS cache yang ditawarkan oleh RouterOS.

1. PENGELOLAAN ARP DI MIKROTIK

Tampilan topologi di gns3

Dari sumber MikroTik Documentation (2023), Address Resolution Protocol (ARP) adalah protokol yang digunakan untuk memetakan alamat Internet Protocol (IP) ke alamat Media Access Control (MAC) pada jaringan lokal. Router memiliki tabel entri ARP yang digunakan saat ini. Secara normal, tabel ini dibangun secara dinamis, tetapi untuk meningkatkan keamanan jaringan, entri statis dapat ditambahkan.

Untuk melihat tabel ARP yang saat ini aktif, gunakan perintah:

/ip arp print
# Menampilkan semua entri ARP

Dari sumber abcXperts (2024), jumlah maksimum entri ARP default bergantung pada jumlah RAM yang terpasang pada perangkat. Nilai ini dapat disesuaikan menggunakan perintah berikut:

/ip settings set max-neighbor-entries=1000
# Mengatur jumlah maksimum entri ARP

Untuk menambahkan entri ARP statis, gunakan perintah:

/ip arp add address=192.168.1.2 mac-address=00:11:22:33:44:55 interface=ether1
# Menambahkan entri ARP statis

Dari sumber MikroTik Documentation (2023), ada beberapa mode ARP yang dapat digunakan pada antarmuka MikroTik, antara lain: enabled (default), disabled, proxy-arp, reply-only, dan local-proxy-arp. Setiap mode memiliki fungsi yang berbeda dalam menangani permintaan dan respons ARP.

Untuk mengaktifkan mode reply-only pada sebuah interface:

/interface ethernet set ether1 arp=reply-only
# Mengaktifkan mode reply-only

Dari sumber abcXperts (2024), ketika mode reply-only diaktifkan, interface hanya akan menjawab permintaan ARP dan alamat MAC tetangga akan diselesaikan hanya menggunakan entri yang dikonfigurasi secara statis dari menu "/ip arp". Mode ini berguna untuk situasi di mana Anda perlu mengontrol secara ketat alamat IP mana yang dapat ditetapkan ke antarmuka tertentu.

Untuk mengaktifkan mode proxy-arp di sebuah interface:

/interface ethernet set ether1 arp=proxy-arp
# Mengaktifkan mode proxy-arp

Dari sumber MikroTik Documentation (2023), fitur Proxy-ARP berarti router akan mendengarkan permintaan ARP yang diterima pada antarmuka yang relevan dan meresponnya dengan alamat MAC-nya sendiri jika permintaan tersebut cocok dengan alamat IP lain dari router. Fitur ini berguna jika Anda ingin menetapkan alamat IP ke klien dial-in (ppp, pppoe, pptp) dari ruang alamat yang sama dengan yang digunakan pada LAN yang terhubung.

Jika fitur ARP dinonaktifkan pada antarmuka (arp=disabled), permintaan ARP dari klien tidak akan dijawab oleh router. Oleh karena itu, entri ARP statis juga harus ditambahkan ke klien.

Untuk melihat entri ARP yang dinamis saja:

/ip arp print dynamic
# Menampilkan entri ARP dinamis

Untuk memfilter entri ARP berdasarkan interface:

/ip arp print where interface=ether1
# Memfilter entri ARP berdasarkan interface
Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa ARP di MikroTik merupakan protokol penting yang dapat dikelola dengan berbagai mode (enabled, disabled, reply-only, proxy-arp, dan local-proxy-arp), memungkinkan administrator jaringan untuk mengontrol bagaimana permintaan dan respons ARP ditangani. Pengelolaan ARP yang tepat membantu meningkatkan keamanan jaringan dengan membatasi atau mengontrol bagaimana perangkat dapat berkomunikasi satu sama lain pada jaringan lokal.

2. LOCAL-PROXY-ARP DI MIKROTIK

Dari sumber MikroTik Documentation (2023), jika properti ARP diatur ke local-proxy-arp pada antarmuka, maka router melakukan proxy ARP ke/dari antarmuka ini saja, yaitu untuk lalu lintas yang masuk dan keluar dari antarmuka yang sama. Dalam LAN normal, perilaku default adalah dua host jaringan berkomunikasi langsung satu sama lain, tanpa melibatkan router.

Untuk mengaktifkan local-proxy-arp pada sebuah interface:

/interface ethernet set ether1 arp=local-proxy-arp
# Mengaktifkan mode local-proxy-arp

Dari sumber abcXperts (2024), dengan local-proxy-arp diaktifkan, router akan merespons semua host klien dengan alamat MAC antarmuka router itu sendiri, bukan alamat MAC host lain. Misalnya, jika Host A (192.168.88.2/24) menanyakan alamat MAC dari Host B (192.168.88.3/24), router akan merespons dengan alamat MAC-nya sendiri. Dengan kata lain, jika local-proxy-arp diaktifkan, router akan bertanggung jawab untuk meneruskan lalu lintas antara Host A dan Host B.

Semua entri cache ARP pada Host A dan Host B akan mereferensikan alamat MAC router. Dalam kasus ini router melakukan local-proxy-arp untuk seluruh subnet 192.168.88.0/24.

Contoh implementasi local-proxy-arp untuk konfigurasi bridge dengan server DHCP dan port bridge yang terisolasi:

/interface bridge add arp=local-proxy-arp name=bridge1
/interface bridge port add bridge=bridge1 horizon=1 interface=ether2
/interface bridge port add bridge=bridge1 horizon=1 interface=ether3
/interface bridge port add bridge=bridge1 horizon=1 interface=ether4
# Konfigurasi bridge dengan local-proxy-arp

Dari sumber MikroTik Forum (2021), opsi "published" pada entri ARP statis di RouterOS dapat digunakan untuk menerapkan proxy-arp secara lebih granular. Ini memungkinkan Anda untuk mengatur entri ARP statis yang akan dipublikasikan, sehingga tidak perlu mengatur proxy-arp pada seluruh antarmuka untuk semua MAC yang akan di-proxy-arp-kan.

/ip arp add address=192.168.1.5 mac-address=00:11:22:33:44:77 interface=ether1 published=yes
# Menambahkan entri ARP statis dengan flag published

Penggunaan ARP proxy dapat meningkatkan keamanan jaringan dengan memungkinkan isolasi host pada level Layer 2 sambil tetap memungkinkan komunikasi pada level Layer 3 melalui IP bridge.

Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa local-proxy-arp adalah fitur penting di MikroTik yang memungkinkan router untuk berperan sebagai perantara dalam komunikasi antar host pada subnet yang sama, dengan merespon semua permintaan ARP menggunakan alamat MAC-nya sendiri. Fitur ini sangat berguna untuk skenario isolasi port bridge, segmentasi jaringan, dan kontrol komunikasi antar host yang berada pada subnet yang sama namun perlu diisolasi pada layer 2.

3. MIKROTIK NEIGHBOR DISCOVERY PROTOCOL (MNDP)

Dari sumber MikroTik Documentation (2023), MikroTik Neighbor Discovery Protocol (MNDP) adalah protokol yang memungkinkan untuk menemukan perangkat lain yang kompatibel dengan MNDP, CDP (Cisco Discovery Protocol), atau LLDP (Link Layer Discovery Protocol) dalam domain broadcast Layer 2. Fitur ini dapat digunakan untuk memetakan jaringan Anda.

Untuk melihat daftar tetangga yang ditemukan:

/ip neighbor print
# Menampilkan semua perangkat tetangga

Contoh output dari perintah di atas akan menampilkan informasi seperti interface, alamat IP/MAC, identitas perangkat, versi RouterOS, dan tipe board dari perangkat tetangga.

# OUTPUT
# INTERFACE ADDRESS MAC-ADDRESS IDENTITY VERSION BOARD
0 ether13 192.168.33.2 00:0C:42:00:38:9F MikroTik 5.99 RB1100AHx2
1 ether11 1.1.1.4 00:0C:42:40:94:25 test-host 5.8 RB1000
2 Local 10.0.11.203 00:02:B9:3E:AD:E0 c2611-r1 Cisco I...

Dari sumber codingpackets.com (2022), LLDP dan protokol discovery lainnya dikonfigurasi di bagian /ip neighbor/discovery-settings/. Anda dapat mengaktifkan atau menonaktifkan protokol yang digunakan untuk discovery:

/ip neighbor discovery-settings set protocol=cdp,lldp,mndp
# Mengatur protokol discovery yang aktif

Dari sumber MikroTik Documentation (2023), dimungkinkan untuk mengubah apakah suatu antarmuka berpartisipasi dalam neighbor discovery atau tidak menggunakan Interface list. Jika antarmuka dimasukkan ke dalam daftar discovery, antarmuka tersebut akan mengirimkan informasi dasar tentang sistem dan memproses paket discovery yang diterima dalam jaringan Layer 2.

Untuk mengonfigurasi interface yang berpartisipasi dalam neighbor discovery:

/interface list add name=discovery-interfaces
/interface list member add interface=ether1 list=discovery-interfaces
/interface list member add interface=ether2 list=discovery-interfaces
/ip neighbor discovery-settings set discover-interface-list=discovery-interfaces
# Mengatur interface untuk neighbor discovery

Dari sumber MikroTik Documentation (2023), sejak RouterOS v6.44, neighbor discovery bekerja pada interface slave individu. Ketika interface master (misalnya bonding atau bridge) dimasukkan dalam daftar interface discovery, semua interface slave-nya akan secara otomatis berpartisipasi dalam neighbor discovery.

Menghapus antarmuka dari daftar antarmuka discovery akan menonaktifkan penemuan tetangga pada antarmuka ini dan juga kemungkinan penemuan perangkat itu sendiri pada antarmuka tersebut, yang dapat berguna untuk keamanan jaringan.

Contoh konfigurasi untuk mengizinkan neighbor discovery hanya pada beberapa interface slave:

/interface bonding add name=bond1 slaves=ether5,ether6
/interface list add name=only-ether5
/interface list member add interface=ether5 list=only-ether5
/ip neighbor discovery-settings set discover-interface-list=only-ether5
# Konfigurasi discovery hanya pada ether5
Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa MikroTik Neighbor Discovery Protocol (MNDP) adalah fitur yang sangat berguna untuk menemukan dan memetakan perangkat jaringan dalam domain broadcast Layer 2. Protokol ini kompatibel dengan CDP dan LLDP, memungkinkan penemuan perangkat dari berbagai vendor. Pengaturan yang tepat dari antarmuka yang berpartisipasi dalam discovery dapat membantu dalam manajemen jaringan dan juga berkontribusi pada keamanan jaringan.
